Agenda

  • Modułowość kodu JS

    nie tylko AMD czy CommonJS, ale ogólnie jak dzielić kod JS, żeby ułatwić jego późniejsze utrzymanie

  • Wzorce projektowe w JS

    Może nawet nie będą to konkretne wzorce (chociaż na pewno omówię module pattern czy ogólnie podejście do dziedziczenia w JS), ale chciałbym też opowiedzieć o typowych zastosowaniach JS i jakich podejść się trzymać (np. eventy w JS, deferred object itp.).

  • Testowanie kodu

    Mogę omówić Jasmine i Mocha (jak bardzo ktoś będzie chciał to i QUnit, chociaż nie używałem).

  • Wydajność aplikacji JS

    uczuliłbym jak działa sam JS i gdzie mogą pojawiać się ‘bottlenecki’, ale do tej pory nie miałem problemów wydajnościowych stricte JS’owych. jak nie zasypać backend’u zbyt dużą request’ów :D.
